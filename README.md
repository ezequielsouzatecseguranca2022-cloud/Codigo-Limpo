# Codigo-Limpo
## Nome Significativo
### A escolha de nomes adequados para variáveis, funções, classes e demais elementos do código é um dos pilares da legibilidade e manutenção de sistemas.
•	Utilizar nomes que revelem claramente a intenção do código.
•	Evitar abreviações desnecessárias ou ambíguas.
•	Adotar nomes fáceis de pronunciar e pesquisar.
•	Manter consistência na nomenclatura utilizada em todo o projeto.
Nomes bem definidos reduzem a necessidade de explicações adicionais e facilitam a compreensão por toda a equipe de desenvolvimento.

##  Comentários
### Comentários devem complementar o código e não servir como substitutos para uma implementação confusa ou mal estruturada.
•	Priorizar código autoexplicativo.
•	Utilizar comentários apenas quando agregarem valor.
•	Remover comentários redundantes ou desatualizados.
•	Documentar regras de negócio e decisões complexas.
Quanto mais claro for o código, menor será a dependência de comentários para explicar seu funcionamento.

##  Formatação
### A formatação adequada melhora significativamente a legibilidade e a organização do software.
•	Manter um padrão visual consistente.
•	Agrupar elementos relacionados.
•	Utilizar espaçamentos adequados para facilitar a leitura.
•	Organizar funções e classes de forma lógica.
Uma boa formatação contribui para a produtividade da equipe e para a qualidade geral do projeto

##  Conclusão
### Os capítulos analisados demonstram que o desenvolvimento de software vai além da criação de funcionalidades. A utilização de nomes significativos, comentários adequados e uma formatação consistente torna o código mais compreensível, e preparado para futuras evoluções. Essas práticas representam investimentos de longo prazo na qualidade do software.
